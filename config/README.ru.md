# kitty config

Мой рабочий, живой конфиг [kitty](https://sw.kovidgoyal.net/kitty/) — macOS.
Оформление в стиле Ghostty, сплиты в стиле Ghostty и скруглённые powerline-табы,
плюс фиксы, чтобы `Cmd`/`Ctrl`-шорткаты работали на русской раскладке.

[English](README.md) · [Русский](README.ru.md)

> Это персональные macOS-биндинги на `Cmd` с кириллическими дублями. **Форкни,
> прочитай, адаптируй** — не тащи всё вслепую. Что менять под свою машину и
> раскладку — в разделе [Как адаптировать](#как-адаптировать).

## Оформление

- **Палитра** — дефолтная тёмная тема Ghostty, снятая через `ghostty
  +show-config --default` (foreground/background + 16 ANSI-цветов), в
  `ghostty.conf`.
- **Шрифт** — JetBrains Mono 12.
- **Табы** — сверху, скруглённые «пилюли» powerline (`tabs.conf`).
- **Сплиты** — панели в стиле Ghostty (`splits.conf`).
- **Титлбар** — под цвет фона терминала (macOS).

## Требования

- **macOS** — используются `macos_titlebar_color`, `hide_macos_app`,
  `minimize_macos_window`, биндинги на `Cmd` и macOS-специфичный фикс клавиатуры
  (см. [Фикс Cmd+W](#фикс-cmdw)).
- **kitty** — проверялось на 0.47.
- **Установленный JetBrains Mono** — иначе kitty молча берёт другой шрифт и вид
  меняется. Свой шрифт — через `font_family` в `ghostty.conf`.
- **Опционально:** киты из этого репозитория — подключаются отдельно командой
  `familiar enable --kittens` (см. [главный README](../README.ru.md)).

## Установка

Простой путь — хелпер `familiar`: `familiar enable --all` подключает этот конфиг
(как `terminal.conf`) и киты. См. [главный README](../README.ru.md).

Использовать только этот конфиг терминала руками — скопируй файлы и перезапусти kitty:

```sh
cp config/*.conf ~/.config/kitty/
```

> `cp` перезаписывает одноимённые файлы (прежде всего `kitty.conf`) — сначала
> сохрани свой текущий конфиг, если он есть.

Тема, сплиты, табы и раскладка заработают сразу. Киты добавляются отдельно
командой `familiar enable --kittens` — она генерирует их мапы с абсолютными
путями, которые нужны kitty.

## Что в каждом файле

Сгруппировано в `keys/` (клавиши + раскладка) и `look/` (внешний вид); два хаба
(`kitty.conf`, `terminal.conf`) остаются в корне.

| Файл | Что задаёт |
|---|---|
| `kitty.conf` | Тонкий вход для ручного `cp` — просто `include terminal.conf`. |
| `terminal.conf` | Хаб «оформления» — подключает остальные по порядку (`restored-defaults` **первым**, чтобы его `clear_all_shortcuts` отработал до всех `map`). Именно его включает `familiar enable --all`. |
| `keys/restored-defaults.conf` | `clear_all_shortcuts` + выверенный набор macOS-`Cmd`: размер шрифта (`Cmd+=/-/0`), новое окно/таб (`Cmd+N/T/Enter`), навигация по табам (`Cmd+Shift+[ ]`), переименование таба (`Cmd+Shift+I`), буфер (`Cmd+C/V`), поиск (`Cmd+F`), очистка экрана (`Cmd+K/L`), скролл, quit/hide/minimize (`Cmd+Q/H/M`), правка/перезагрузка конфига (`Cmd+,` / `Cmd+Ctrl+,`). |
| `look/ghostty.conf` | Оформление: дефолтная палитра Ghostty + JetBrains Mono 12 + титлбар macOS под фон. **Нужен установленный шрифт.** |
| `keys/russian-ctrl.conf` | Ремапит `Ctrl+<кириллица>` в тот же control-код, что латинская клавиша, по физической позиции ЙЦУКЕН→QWERTY — чтобы `Ctrl+C/V/…` работали при наборе на русском. |
| `keys/splits.conf` | Панели в стиле Ghostty: `Cmd+D` / `Cmd+Shift+D` — сплит, `Cmd+Shift+стрелки` — фокус, `Cmd+Shift+Enter` — зум (stack), `Cmd+W` — закрыть панель, `Cmd+Shift+W` — закрыть таб. |
| `look/tabs.conf` | Скруглённые powerline-«пилюли» сверху + цвета, `Cmd+1…9` — переход к табу. |
| `look/darcula.conf` | Необязательная тема JetBrains Darcula (`familiar enable --theme darcula`). Грузится **после** `ghostty.conf` и `tabs.conf`, поэтому переопределяет их цвета (палитра терминала и панель табов); шрифт, титлбар и форма «пилюль» остаются. |

Важно: `clear_all_shortcuts` снимает **все** встроенные шорткаты kitty, включая
кросс-платформенные `Ctrl+Shift` (`kitty_mod`) — обратно возвращается только
выверенный набор на `Cmd`. `restored-defaults.conf` — снимок дефолтов kitty,
замороженный на версии; после мажорного апгрейда стоит перепроверить.

## Русская раскладка

Хоткеи работают при активной кириллице за счёт двух вещей:

- `russian-ctrl.conf` мапит каждый `Ctrl+<кириллица>` в control-код латинской
  клавиши той же физической позиции (`Ctrl+с` → `Ctrl+C`, …).
- `splits.conf` несёт кириллические дубли `Cmd`-биндингов (`D→в` для сплита);
  дубли китов (`S→ы`, `R→к`, `L→д`) генерирует `familiar`. Переключение табов —
  на цифрах (`Cmd+1…9`), они не зависят от раскладки, дубли не нужны.

**Не пользуешься русской раскладкой?** Удали `russian-ctrl.conf` (и его `include`
из `terminal.conf`) и убери кириллические дубли `map` — латинские биндинги не
затронуты.

## Фикс Cmd+W

Единственная по-настоящему неочевидная часть. На macOS встроенный
`Cmd+W → close_tab` у kitty «липкий»: у него есть ASCII-fallback, который заранее
конвертит физическую **W** — а она на русской раскладке даёт `ц` — обратно в `w`
и перехватывает событие, из-за чего `Cmd+W` закрывал *таб*, а не панель, даже на
кириллице.

- `clear_all_shortcuts yes` — единственный способ снять этот дефолт; затем
  `restored-defaults.conf` возвращает только нужные мне `Cmd`-шорткаты.
- `splits.conf` мапит `Cmd+W` на `close_window` с `--allow-fallback=ascii`: он
  переводит физическую W в латинскую `w` и надёжно матчится на любой раскладке.
- Прямой `map cmd+ц` **не** работает — macOS input method его съедает (проверено
  через `kitty --debug-keyboard`). ASCII-fallback — надёжный путь.

`--allow-fallback=ascii` срабатывает только для не-ASCII ввода, поэтому латинские
раскладки (Dvorak, Colemak, …) не затрагиваются.

## Как адаптировать

- **Не macOS?** Убери `macos_*` из `ghostty.conf` и переназначь `Cmd` на
  `Ctrl`/`Super` везде; фикс `Cmd+W` — только для macOS.
- **Нет русской раскладки?** См. [Русская раскладка](#русская-раскладка).
- **Другой шрифт?** Поменяй `font_family` в `ghostty.conf`.
- **Только терминал?** `familiar enable --terminal` подключает только этот
  конфиг, без китов (или ручной `cp` выше).

## Своя тема

Тема — это два одноимённых файла. Допустим, нужен `nord`:

1. **`look/nord.conf`** — сторона терминала: `foreground`, `background`, `cursor`,
   `selection_background`, `color0`…`color15` и, если нужно, цвета табов. Файл
   подключается *после* `ghostty.conf` и `tabs.conf`, поэтому достаточно перечислить
   только то, что меняется.
2. **`palette/nord.conf`** — сторона китов: подсветка синтаксиса и фоны диффа,
   по строке `роль значение`; значение — либо номер 256-цветной палитры, либо
   `#rrggbb` (рисуется truecolor). Пропущенная роль наследует цвет дефолтной темы
   (`ghostty`). Полный список ролей — в `palette/darcula.conf`.

Никаких списков обновлять не нужно: и флаг `--theme`, и киты находят темы сканом
`config/palette/` (двумя независимыми сканами — CLI не должен импортировать пакет
китов, — поэтому их согласованность стережёт тест).

Дальше `familiar enable --all --theme nord`. Терминал подхватит цвета перезагрузкой
конфига, а киты читают `FAMILIAR_THEME` при старте — им нужен перезапуск kitty.
