## [0.21.0] — 2026-07-16

### Changed

- review: `Shift+клик` / `Shift+↑↓` в дереве файлов работают как классические
  GUI-списки — диапазон красится от неподвижного якоря (файла в фокусе при
  входе в мультивыбор), и повторное растягивание обратно к якорю снимает
  метки с ушедших из диапазона строк. `Shift+↑/↓` перескакивает строки без
  вклада в диапазон (развёрнутые папки) — пустых шагов нет. Точечное снятие
  метки с файла или папки — `⌥+клик`.
- review: при активном мультивыборе выделение показывают только метки;
  строка курсора без метки рисуется как обычная — так `⌥+клик` наглядно
  снимает выделение с текущего файла. Первый `⌥+клик` забирает в выборку и
  активный файл (как `⌘+клик` в GUI-списках) — получается два выделенных
  файла, а не переехавшая подсветка. Выделение в дереве не пустеет:
  `⌥+клик` по последней помеченной строке оставляет её помеченной (снятие —
  обычная навигация или Esc), и клики метками не переключают открытый дифф.

### Fixed

- review: `⌥+клик` по строке дерева больше не переносит курсор и не уводит
  фокус из диффа — только переключает метку, текущий просматриваемый файл
  остаётся выбранным.

## [0.20.0] — 2026-07-16

### Added

- review: **множественный выбор в дереве файлов.** `Shift+↑/↓` красит диапазон
  файлов от курсора; `Shift+клик` дотягивает выделение до кликнутой строки
  (как в GUI-списках — файл под курсором остаётся выделенным); `⌥+клик`
  точечно помечает/снимает файл или папку целиком, а `Shift+клик` по уже
  помеченной строке снимает метку. Диапазон помечает ровно
  то, что видно: развёрнутая папка сама не добавляет ничего (её файлы — свои
  строки), свёрнутая добавляет все свои файлы и подсвечивается сама.
  Помеченные строки выглядят
  как обычное выделение; обычная навигация (стрелки, клик, Home/End, `g`/`G`)
  или Esc снимает метки. `⌘c` копирует все помеченные пути как `@path`-
  упоминания, по одному на строку — готово для вставки в промпт Claude.
- конфиг терминала: **Shift+клик пробрасывается китам.** kitty обрабатывает
  `shift+left` сам, даже когда программа захватила мышь, — киты его не видели;
  `config/keys/mouse.conf` снимает эти grabbed-маппинги. Цена: внутри программ,
  захватывающих мышь (vim с мышью, htop), Shift+drag больше не запускает
  собственное выделение kitty — обычный шелл не затронут.

## [0.19.0] — 2026-07-15

### Added

- **Комментарии ревью — сразу в промпт Claude.** В `review` клавиша `s`
  собирает все комментарии в markdown — как `w`, — но вместо буфера обмена
  закрывает оверлей и вставляет текст прямо в окно, поверх которого он был
  открыт (там обычно ждёт Claude). Вставка bracketed, поэтому многострочное
  ревью ложится одним блоком и не отправляется переводами строк. `w`
  (копирование в буфер) работает как раньше.

### Changed

- **Esc на верхнем уровне теперь спрашивает перед закрытием.** В `log`,
  `review` и `session` Esc на дне каскада вместо тихого «ничего» показывает
  центрированный диалог в стиле kitty с кнопками Yes/No: `y`/`Enter` закрывает
  оверлей, `n`/`Esc` оставляет открытым, ←/→/Tab переключают кнопки, по кнопкам
  можно кликнуть (при наведении — курсор-рука). `q` и `⌃c` по-прежнему выходят
  сразу — `⌃c` работает даже при открытом диалоге.

## [0.18.0] — 2026-07-14

### Changed

- **Esc больше не закрывает оверлеи китов.** В `log`, `review` и `session` Esc
  по-прежнему ведёт по каскаду назад (выделение → поиск → фильтр → экран), но
  на верхнем уровне ничего не делает вместо выхода — случайный Esc не смахнёт
  оверлей. Выход — `q` или `⌃c`. Бонусом Esc на верхнем уровне `log` и `review`
  теперь сбрасывает применённый фильтр.

## [0.17.0] — 2026-07-14

### Added

- **Уведомление об обновлении.** Раз в сутки кит проверяет на GitHub выход новой
  версии (фоновым потоком, кэш в `~/.cache/familiar/update.json`) и разово
  показывает подсказку в футере: `familiar X.Y.Z is out — brew upgrade familiar`.
  `familiar status` теперь печатает установленную версию и последнюю доступную.
  `FAMILIAR_UPDATE_CHECK=0` выключает и проверку, и подсказку. Номер версии
  теперь живёт в файле `VERSION` в корне репозитория — единый источник для CLI
  и китов.

## [0.16.0] — 2026-07-14

### Changed

- **Таб-бар упрощён до `номер: проект❘тема`.** Символ звонка 🔔 и процент
  прогресса OSC 9;4 из 0.15.0 убраны: процент всё равно не рисовался (kitty
  0.47 трактует этот escape как уведомление), а уведомления лучше обрабатывать
  вне терминала.

## [0.15.0] — 2026-07-14

### Changed

- **Умные заголовки табов для работы с Claude Code** (`look/tabs.conf`). Заголовки
  ограничены 30 ячейками, чтобы в бар помещалось несколько сессий; таб с Claude Code
  (узнаётся по `✳ `, которую CLI ставит в титул окна) показывает `проект❘тема`
  вместо сырого титула, обычные шелл-табы не трогаются. Красный 🔔 отмечает фоновый
  таб со звонком (terminal-bell уведомление Claude), а процент прогресса задачи
  появляется, когда программа его сообщает (OSC 9;4).

## [0.14.0] — 2026-07-13

### Changed

- **Документация переехала в [GitHub wiki](https://github.com/DenoBY/familiar/wiki).**
  Папки `docs/` (страницы китов и скриншоты) больше нет ни в репозитории, ни в
  Homebrew-поставке; README ссылается на wiki. Wiki двуязычная (английские и русские
  страницы).

### Fixed

- **`⌃o` (возврат после прыжка к определению) работает на русской раскладке.**
  Терминальный конфиг мапит `ctrl+<кириллица>` в C0-байты (`send_text`), поэтому на
  ЙЦУКЕН сочетание приходит текстом, а не key-событием — просмотрщик review теперь
  разбирает эти байты для всех своих ctrl-шорткатов (`⌃o` назад, `⌃u`/`⌃d` скролл,
  `⌃w`/`⌃u` при наборе комментария).
- **Переход к определению стал быстрым на больших репозиториях.** Поиск определения
  теперь грепает только файлы языка клика (с fallback на repo-wide проход по
  исходникам), так что тяжёлые untracked-логи и минифицированные бандлы больше не
  подвешивают вьювер — на реальном Laravel-репозитории прыжок ускорился с ~3 с
  замершего UI до <0.1 с.

### Added

- **Переход к определению понимает синтаксис PHP и Ruby.** `$this->method()`,
  `Order::create()`, `?->`, `static::` и рубишный `Foo::Bar` теперь резолвят
  получателя: методы `$this`/`static` ищутся сперва в текущем файле, а вызов через
  импортированный класс (`use App\Models\Order; Order::create()`) прыгает сразу в
  файл этого класса. Тот же резолв получателя заработал для импортов Python
  (`mod.func`) и JS (`Api.get`); объявления PHP 8.1 `enum` распознаются.
- **Генератор демо-стенда** (`tools/demo_stand.py`): собирает демонстрационный
  git-репозиторий с фотогеничной историей, ветками и незакоммиченными правками плюс
  фейковое хранилище Claude Code с живыми сессиями — воспроизводимая сцена, чтобы
  переснимать скриншоты wiki после изменений UI.

## [0.13.0] — 2026-07-11

### Added

- **Переход к определению в просмотрщике review.** ⌥-клик по символу в диффе — или выделить
  слово и нажать `d` — прыгает к месту его объявления. Прыжок происходит внутри
  просмотрщика со стеком «назад» (`⌃o` — вернуться): определение в изменённом файле
  открывает его дифф, определение в неизменённом — открывается read-only. Если у символа
  несколько определений, их показывает пикер (`1`–`9` или клик). Резолв учитывает контекст:
  `obj.name` предпочитает методы, `name(` — объявления, а импортнутые имена ведут в точный
  файл по импортам текущего файла (Python `from a.b import x` и относительные `.`/`..`;
  JS/TS `import`/`require` с расширениями и `index`; PHP `use` через composer PSR-4; символы
  Go-пакетов через `go.mod`). Всё прочее — repo-wide `git grep`, включая новые
  untracked-файлы, без индекса и языкового сервера. ⌘-клик невозможен: мышиный протокол
  терминала несёт только Shift/Alt/Ctrl, но не Cmd, поэтому триггер — ⌥-клик.
- **Мышиные жесты в диффе review.** Двойной клик по слову выделяет именно слово (для
  копирования), а клик по номеру строки ставит к ней комментарий — хоткей `c`/Enter тоже
  работает. Указатель превращается в руку над номерами строк и, пока зажат ⌥, над
  идентификаторами, к которым можно перейти. Выделение текста теперь сохраняет подсветку
  синтаксиса, добавляя только фон.

## [0.12.0] — 2026-07-10

### Added

- **Форма указателя мыши.** В просмотрщике диффов (review, log) и в предпросмотре сессии
  указатель теперь отражает то, что под ним: текстовый курсор (I-beam) над кодом и прочим
  выделяемым текстом и рука над кликабельным — папками в дереве файлов и свёрнутыми
  gap-разделителями в диффе (review, log), сворачиваемыми записями в предпросмотре сессии.
  В остальных местах остаётся стрелка. Раньше включённый mouse-tracking всюду навязывал
  стрелку, скрывая, где можно выделять, а где кликать.

### Changed

- **Тема — это два data-файла, без Python.** Раньше требовались словарь-палитра в
  `plugins/modules/theme.py` и кортеж `THEMES` в `bin/familiar`. Теперь тема — это
  `config/look/<имя>.conf` (цвета терминала) плюс `config/palette/<имя>.conf` (подсветка
  в китах, по строке `роль значение`). И флаг `--theme`, и киты находят темы сканом
  `config/palette/`, поэтому списки нигде не нужно держать синхронными, а пропущенная роль
  наследует цвет дефолтной темы `ghostty`. См. `config/README.md`.

### Fixed

- log: неудачный `fetch` или `push` мог показать чужую ошибку. Фоновый сетевой вызов и
  git-команды главного потока писали в один общий слот ошибки, поэтому прокрутка списка
  коммитов во время push могла затереть сообщение до того, как его показали. Сетевые вызовы
  теперь возвращают ошибку напрямую.

### Internal

- Проход по стилю китов согласно своду проекта: предпросмотр сессии вынесен в
  `modules/session/preview.py`, общая обработка клавиш diff-панели review и log поднята в
  `DiffTreeView`, логика указателя вынесена в миксин `PointerCursor`, в хендлерах
  проставлены type hints, документация пересинхронизирована с кодом.

## [0.11.0] — 2026-07-10

### Added

- log: `p` — `git push` текущей ветки. Спрашивает подтверждение, как `-` в review: в футере
  видно, сколько коммитов уедет и куда (`push 3 commits to origin/main?`), и подтверждает
  только `y` — промах по клавише ничего не публикует. Ветку без upstream создаёт на
  `origin` (`push -u`) и привязывает к нему, как это делает IDE. Подсказка `p push` видна,
  только пока есть незапушенные коммиты; сеть уходит в фон, UI не замирает.

## [0.10.0] — 2026-07-10

### Changed

- Тема по умолчанию называется `ghostty` — по палитре, которой она и является, а не
  `default`. Вернуться к ней: `familiar enable --theme ghostty`. Старое имя `--theme
  default` больше не принимается.
- Тема Darcula перекрашивает не только палитру терминала, но и панель табов и фоны
  диффа внутри китов. Как собрать свою тему — в `config/README.ru.md`.
- log: счётчик в шапке показывает `300+`, пока история дочитана не до конца. Раньше он
  печатал число загруженных коммитов без пометки, и `(300)` читалось как «в ветке всего
  300 коммитов».

### Fixed

- log: при быстрой прокрутке список отвечал рывками — панель подробностей на каждый шаг
  курсора звала git (`show` + `branch --contains`, а тот обходит все ветки репозитория).
  Теперь подробности подтягиваются после паузы в прокрутке, как дифф в дереве файлов;
  пока они грузятся, панель показывает то, что уже есть в списке.
- Текст терминала в Darcula отдавал синевой: брался редакторский `TEXT` (`#a9b7c6`), тогда
  как для консоли схема JetBrains задаёт отдельный `CONSOLE_NORMAL_OUTPUT` (`#bbbbbb`).
- Darcula: подсветка изменившихся слов тонула в фоне строки. Фон строки затемнён, слово
  остаётся насыщенным — контраст между ними вырос вдвое, а текст поверх читается лучше.
- review, log: клик по папке в дереве сразу сворачивал её, поэтому промах на строку
  перестраивал дерево под курсором. Теперь первый клик выделяет папку, второй по ней же —
  сворачивает, как это давно делает список коммитов в `log`.

### Internal

- `ruff` не проверял `bin/familiar` (файл без расширения `.py`) — в нём накопилось 22
  нарушения лимита строки. Теперь проверяет, и `ruff check .` стоит в CI.
- Тест сверяет список тем в CLI со списком палитр в китах: разойдись они, `--theme` принял
  бы имя и молча покрасил дефолтной палитрой.

## [0.9.0] — 2026-07-10

### Added

- **Цветовые темы** (`familiar enable --theme darcula`): палитра JetBrains Darcula для
  терминала *и* для подсветки синтаксиса внутри китов. Цвета взяты из официальной схемы
  `Darcula` в JetBrains/intellij-community, поэтому ключевые слова, строки, числа,
  докстринги и декораторы попадают ровно в те оттенки, которыми их красит IDE. Без
  `--terminal`/`--all` перекрашиваются только киты — внешний вид терминала familiar не
  трогает, пока его об этом не попросят. `--theme default` оставляет прежний вид, а
  `familiar status` показывает активную тему.
- Киты рисуют truecolor, когда тема того требует (`kitty.fast_data_types.Color`), а не
  огрубляют каждый оттенок до 256-цветного куба.

## [0.8.0] — 2026-07-10

### Removed

- review: git-скоупы **staged** и **vs \<ветка\>**, а вместе с ними клавиша `s`.
  Кит всегда показывает незакоммиченные правки рабочего дерева (vs `HEAD`) —
  единственный скоуп, которым пользовались. `+` по-прежнему делает `git add`.

### Fixed

- log: футер предлагал `a current branch`, когда ты и так на текущей ветке — теперь
  подсказка называет режим, *в который* переключит `a`, как и соседние подсказки.

## [0.7.0] — 2026-07-10

### Added

- review, log: вид **финального кода** (`v`) — файл таким, каким его показывает IDE после
  слияния: без знаков `+`/`−`, без удалённых строк и без заливки внутри строк. Правки
  видны маркером на полях (`▎` зелёный — добавлено, `▎` синий — изменено, `▔` красный —
  здесь код вырезан). Прыжки, поиск, комментарии и копирование работают как в диффе, а
  курсор при переключении остаётся на той же строке.
- review, log: **карта изменений** на полосе прокрутки справа от диффа — цветные риски
  показывают, где правки, и в длинном файле сразу видно, куда листать.
- **Подсветка синтаксиса уровня IDE** на Pygments: функции, типы, `self`, декораторы,
  докстринги и f-строки вместо прежних строк/комментариев/чисел/ключевых слов. Pygments
  лежит рядом, в `plugins/vendor`, так что ставить по-прежнему нечего; если его нет,
  работает встроенный регексп-лексер.

### Changed

- Flash-сообщения над футером гаснут сами через 2.5 с — подсказки футера возвращаются, не
  дожидаясь следующего нажатия клавиши.

### Fixed

- review, log: когда блок строк заменялся меньшим числом строк, на полях стоял только
  `▎ изменено`, а удаление ничем не отмечалось; теперь `▔` ставится там, где код вырезан.
- Цвета подсветки больше не съезжают на строку ниже form feed, одинокого `\r` или U+2028 —
  карта цветов режется ровно по тем разделителям, что использует `str.splitlines()`.

## [0.6.0] — 2026-07-10

### Added

- review: `-` откатывает файл или папку под курсором к `HEAD` — и рабочее дерево, и
  индекс. У неотслеживаемых файлов восстанавливать нечего, поэтому они удаляются с
  диска; в вопросе-подтверждении это сказано прямо. Ничего не происходит, пока не нажат
  `y`: `Enter`, `Esc` и любая другая клавиша отменяют.

## [0.5.0] — 2026-07-10

### Added

- review: неотслеживаемые файлы собраны в группу **Unversioned Files** внизу дерева,
  свёрнутую по умолчанию, — куча новых файлов больше не хоронит правки, ради которых
  открывали ревью.
- review: `+` добавляет в индекс то, что под курсором, — файл, целую папку или сразу все
  неотслеживаемые (`+` на узле группы). Подсказка появляется, только когда `git add`
  реально что-то сделает; скрытые «шумные» папки не добавляются.
- review, log: **полосы прокрутки** в обеих панелях. Колесо теперь скроллит панель под
  курсором, а не двигает выделение (и не перезагружает дифф на каждый щелчок); стрелки
  по-прежнему двигают выделение и возвращают окно к курсору.
- review: **многострочные комментарии** — `Shift+Enter` вставляет перенос, длинный текст
  переносится по словам, поле ввода растёт до трети экрана.
- review, log: `Ctrl+W` стирает слово перед курсором, `Ctrl+U` — весь текст (привычки
  readline). Пока открыт ввод, они больше не скроллят дифф.
- review: `u` сообщает, сколько ignored-файлов показал или скрыл, — полезно, когда все
  они уезжают внутрь свёрнутой группы Unversioned Files.

### Changed

- review, log: `⌘c` и `⌘shift+c` кладут в буфер **@-упоминание с путём от корня
  репозитория** (`@plugins/review.py`, `@plugins/review.py#L42`, `@plugins/review.py#L42-58`
  для выделенного диапазона, `@plugins/` для папки) вместо абсолютного `путь:строка`.
  Это тот вид, какого ждёт Claude Code: он разрешает `@путь` относительно директории,
  из которой запущен.
- review: `w` очищает замечания после копирования в буфер — выгруженное ревью живёт в
  буфере обмена, а маркеры `●` только мешали следующему проходу.

### Fixed

- review, log: счётчик файлов в шапке считал *строки* дерева, поэтому файлы внутри
  свёрнутых папок (и внутри новой группы Unversioned Files) в него не попадали. В проекте,
  где все ignored-файлы неотслеживаемые, нажатие `u` выглядело так, будто ничего не
  произошло.

## [0.3.0] — 2026-07-09

### Added

- session: предпросмотр диалога стал транскриптом в стиле Claude Code — вызовы
  инструментов показывают аргумент (`⏺ Bash(git status)`), под ними вывод
  строками `⎿`. Пути внутри проекта показываются относительно него. Реплики
  пользователя помечены `>` и плашкой фона на всю ширину — вопрос виден сразу.
- Предпросмотр session: правки файлов рисуются как в Claude Code —
  `⏺ Update(tests/x.py)`, сводка `Added N lines, removed M lines` и diff с
  номерами строк из `structuredPatch` записи (техническое «file updated
  successfully» больше не показывается); изменённые строки — с тем же фоном, что
  в diff-панели `review`, word-diff подсвечивает, что именно поменялось внутри
  строки, diff не сворачивается, а `Write` сообщает число записанных строк.
- Предпросмотр session: ответы Claude рендерят markdown — жирный, курсив,
  инлайн-код, заголовки, списки и fenced-блоки с подсветкой синтаксиса (тем же
  лексером, что и diff-панели; вынесен в общий `modules.highlight`).
- Предпросмотр session: markdown-таблицы рисуются рамкой (линейка между каждой
  строкой, ячейки переносятся по ширине), а не схлопываются в абзац из палок.
- Предпросмотр session: выход из режима планирования рисуется как
  `⏺ Updated plan`, сам план — в рамке (markdown, подсветка кода), под ним
  `⎿ Plan approved` / `⎿ Plan rejected`; чтение файла сворачивается в
  `⎿ Read 402 lines`, а запуск субагента — в
  `⎿ Done (1 tool use · 25.5k tokens · 18s)`.
- Предпросмотр session: длинный вывод, планы и содержимое файлов свёрнуты и
  сообщают об этом (`… +121 lines (ctrl+o to expand)`) — клик раскрывает одну
  запись, `Ctrl+o` / `Enter` — весь свёрнутый вывод разом.
- Предпросмотр session: выделение мышью (кусок строки или строки целиком) и
  копирование по `⌘c`.
- Предпросмотр session: `[` / `]` — прыжки к предыдущей / следующей реплике
  пользователя: промты работают оглавлением длинного диалога.
- session: фоновые агенты (`kind: bg` в реестре живых процессов) помечены `◆` и
  статусом `bg idle` / `bg busy`. К живому агенту Claude Code подключиться не
  даёт, поэтому `o` / `Enter` по такой сессии больше не запускают заведомо
  падающий `claude --resume`, а подсказывают: остановить агента, подключиться
  через `claude agents` или форкнуть диалог (`f` работает — форку живой процесс
  не мешает).

### Fixed

- Предпросмотр session: упавший вызов инструмента больше не выглядит успешным —
  ошибки красным, а вывод не режется до 200 символов в одну строку.
- Предпросмотр session: в репликах пользователя больше не видно служебных
  обёрток (`<local-command-caveat>`, `<system-reminder>`, теги команд) и отчётов
  фоновых задач (`<task-notification>` — килобайты JSON, которых он не писал).
- Предпросмотр session: вставленные картинки больше не отдельная реплика с путём
  до кэша — они висят под своим промтом строками `⎿ [Image #13]`, как в Claude Code.
- review/log: выделение мышью внутри строки теперь включает символ под
  курсором — последний символ больше не теряется при копировании.
- review/log/session: сочетания с модификаторами (`Ctrl+c`, `Ctrl+d` / `Ctrl+u`,
  `⌘c`, `⌘⇧c`) теперь работают на русской раскладке. Раньше они сверялись по
  символу, и на ЙЦУКЕН `⌘с` не делал ничего; теперь раскладка приводится в общем
  `modules.keylayout.chord`, который заодно сверяет весь набор модификаторов —
  `⌘c` больше не срабатывает на `⌘⇧c`.

## [0.2.0] — 2026-07-09

### Added

- `familiar enable --terminal` без имён китов — режим «только терминал».
- `familiar --version`.
- Fallback `$XDG_CONFIG_HOME/kitty` при поиске каталога конфига kitty
  (как у самого kitty).
- CI (GitHub Actions): тесты + `brew style`.

### Changed

- Отрисовка без мигания: кадр применяется атомарно через synchronized
  output kitty (mode 2026).
- Плавная быстрая прокрутка дерева файлов: дифф грузится, когда прокрутка
  утихла, а не на каждый шаг колеса.
- Быстрее диффы больших файлов: дорогой разбор считается один раз на файл
  и переиспользуется при горизонтальном скролле и раскрытии гэпов.
- Бинарные файлы показывают заглушку `(binary file)` вместо нечитаемого
  диффа.
- При ошибке git (`index.lock`, битый репозиторий) киты показывают текст
  ошибки, а не пустое «no changes»/«no commits».

### Fixed

- review: branch-скоуп диффует против merge-base — чужие коммиты базовой
  ветки больше не выглядят «обратными» изменениями.
- review: статистика `+/−` больше не пропадает у переименованных файлов
  в подкаталогах.
- review: staged-скоуп работает в репозитории без коммитов; определение
  базовой ветки понимает имена со слэшем (`release/1.0`).
- log: `git fetch` больше не замораживает UI (работает в фоне).
- session: заголовки сессий и диффы читаются как UTF-8 независимо от
  локали; переименование сессии безопасно при не-ASCII содержимом.
- `familiar disable --restore` восстанавливает kitty.conf из `.bak` и при
  уже снятом managed-блоке.
- Конфликтующие комбинации `familiar enable` (`--all` с именами или
  `--kittens`) — ошибка вместо молчаливого игнорирования.
- Запись kitty.conf стала атомарной (temp-файл + `os.replace`).
- `familiar status` определяет терминал-режим по точной строке `include`.
