## [0.3.0] — 2026-07-09

### Added

- session: предпросмотр диалога стал транскриптом в стиле Claude Code — вызовы
  инструментов показывают аргумент (`⏺ Bash(git status)`), под ними вывод
  строками `⎿`. Пути внутри проекта показываются относительно него. Реплики
  пользователя помечены `>` и плашкой фона на всю ширину — вопрос виден сразу.
- Предпросмотр session: правки файлов рисуются как в Claude Code —
  `⏺ Update(tests/x.py)`, сводка `Added N lines, removed M lines` и diff с
  номерами строк из `structuredPatch` записи (техническое «file updated
  successfully» больше не показывается); изменённые строки — с тем же фоном, что
  в diff-панели `review`, word-diff подсвечивает, что именно поменялось внутри
  строки, diff не сворачивается, а `Write` сообщает число записанных строк.
- Предпросмотр session: ответы Claude рендерят markdown — жирный, курсив,
  инлайн-код, заголовки, списки и fenced-блоки с подсветкой синтаксиса (тем же
  лексером, что и diff-панели; вынесен в общий `modules.highlight`).
- Предпросмотр session: markdown-таблицы рисуются рамкой (линейка между каждой
  строкой, ячейки переносятся по ширине), а не схлопываются в абзац из палок.
- Предпросмотр session: выход из режима планирования рисуется как
  `⏺ Updated plan`, сам план — в рамке (markdown, подсветка кода), под ним
  `⎿ Plan approved` / `⎿ Plan rejected`; чтение файла сворачивается в
  `⎿ Read 402 lines`, а запуск субагента — в
  `⎿ Done (1 tool use · 25.5k tokens · 18s)`.
- Предпросмотр session: длинный вывод, планы и содержимое файлов свёрнуты и
  сообщают об этом (`… +121 lines (ctrl+o to expand)`) — клик раскрывает одну
  запись, `Ctrl+o` / `Enter` — весь свёрнутый вывод разом.
- Предпросмотр session: выделение мышью (кусок строки или строки целиком) и
  копирование по `⌘c`.
- Предпросмотр session: `[` / `]` — прыжки к предыдущей / следующей реплике
  пользователя: промты работают оглавлением длинного диалога.
- session: фоновые агенты (`kind: bg` в реестре живых процессов) помечены `◆` и
  статусом `bg idle` / `bg busy`. К живому агенту Claude Code подключиться не
  даёт, поэтому `o` / `Enter` по такой сессии больше не запускают заведомо
  падающий `claude --resume`, а подсказывают: остановить агента, подключиться
  через `claude agents` или форкнуть диалог (`f` работает — форку живой процесс
  не мешает).

### Fixed

- Предпросмотр session: упавший вызов инструмента больше не выглядит успешным —
  ошибки красным, а вывод не режется до 200 символов в одну строку.
- Предпросмотр session: в репликах пользователя больше не видно служебных
  обёрток (`<local-command-caveat>`, `<system-reminder>`, теги команд) и отчётов
  фоновых задач (`<task-notification>` — килобайты JSON, которых он не писал).
- Предпросмотр session: вставленные картинки больше не отдельная реплика с путём
  до кэша — они висят под своим промтом строками `⎿ [Image #13]`, как в Claude Code.
- review/log: выделение мышью внутри строки теперь включает символ под
  курсором — последний символ больше не теряется при копировании.
- review/log/session: сочетания с модификаторами (`Ctrl+c`, `Ctrl+d` / `Ctrl+u`,
  `⌘c`, `⌘⇧c`) теперь работают на русской раскладке. Раньше они сверялись по
  символу, и на ЙЦУКЕН `⌘с` не делал ничего; теперь раскладка приводится в общем
  `modules.keylayout.chord`, который заодно сверяет весь набор модификаторов —
  `⌘c` больше не срабатывает на `⌘⇧c`.

## [0.2.1] — 2026-07-09

### Changed

- `familiar status` показывает `wired root:` — установку, запечённую в
  `familiar.conf` (код, который kitty реально исполняет), рядом с `app root:`
  вызванной копии, и предупреждает при расхождении.

### Fixed

- `familiar status` больше не сообщает `terminal: no` о терминал-конфиге,
  подключённом из другой установки (детекция зависела от пути вызванной копии).

## [0.2.0] — 2026-07-09

### Added

- `familiar enable --terminal` без имён китов — режим «только терминал».
- `familiar --version`.
- Fallback `$XDG_CONFIG_HOME/kitty` при поиске каталога конфига kitty
  (как у самого kitty).
- CI (GitHub Actions): тесты + `brew style`.

### Changed

- Отрисовка без мигания: кадр применяется атомарно через synchronized
  output kitty (mode 2026).
- Плавная быстрая прокрутка дерева файлов: дифф грузится, когда прокрутка
  утихла, а не на каждый шаг колеса.
- Быстрее диффы больших файлов: дорогой разбор считается один раз на файл
  и переиспользуется при горизонтальном скролле и раскрытии гэпов.
- Бинарные файлы показывают заглушку `(binary file)` вместо нечитаемого
  диффа.
- При ошибке git (`index.lock`, битый репозиторий) киты показывают текст
  ошибки, а не пустое «no changes»/«no commits».

### Fixed

- review: branch-скоуп диффует против merge-base — чужие коммиты базовой
  ветки больше не выглядят «обратными» изменениями.
- review: статистика `+/−` больше не пропадает у переименованных файлов
  в подкаталогах.
- review: staged-скоуп работает в репозитории без коммитов; определение
  базовой ветки понимает имена со слэшем (`release/1.0`).
- log: `git fetch` больше не замораживает UI (работает в фоне).
- session: заголовки сессий и диффы читаются как UTF-8 независимо от
  локали; переименование сессии безопасно при не-ASCII содержимом.
- `familiar disable --restore` восстанавливает kitty.conf из `.bak` и при
  уже снятом managed-блоке.
- Конфликтующие комбинации `familiar enable` (`--all` с именами или
  `--kittens`) — ошибка вместо молчаливого игнорирования.
- Запись kitty.conf стала атомарной (temp-файл + `os.replace`).
- `familiar status` определяет терминал-режим по точной строке `include`.
