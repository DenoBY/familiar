# log

[English](../en/log.md) · [Русский](log.md)

Kitten для [kitty](https://sw.kovidgoyal.net/kitty/): оверлей просмотра истории git —
экран списка коммитов (текущая ветка или все ветки, с графом веток) и по выбранному
коммиту его изменения в двухпанельном виде (дерево файлов + unified diff), как в
[review](./review.md), с подсветкой синтаксиса, поиском и копированием в буфер для
вставки в промт Claude.

Двухпанельная diff-механика — тот же базовый класс, что у review; здесь добавлен экран
списка коммитов, граф веток и панель подробностей.

![log — список коммитов с графом веток и панелью подробностей](../img/log.png)

## Что умеет

- **Список коммитов** текущей ветки; переключатель `a` — показать **все ветки** (`--all`).
- **Граф веток** слева (тумблер `g`) — цветные лейны слияний/ответвлений, как в IDE.
- **Панель подробностей** справа (тумблер `i`): полное сообщение коммита, автор, почта,
  дата, коммиттер (если отличается) и список веток, в которых есть коммит.
- **Ref-метки** в строке: локальные ветки, `origin & name` (схлопнутая пара локаль+удалёнка),
  удалённые и теги — каждая своим цветом.
- **`git fetch`** прямо из оверлея (`f`) — подтянуть удалёнки и перечитать список.
- **Копировать hash** выбранного коммита (`⌘c`) — для вставки в промт/команду.
- **Ленивая подгрузка**: коммиты тянутся пачками по мере докрутки вниз.
- **Фильтр** (`/`) — по subject, короткому hash и автору.
- По выбранному коммиту (`Enter` / `→`) — его изменения относительно первого родителя:
  дерево файлов + unified diff с подсветкой синтаксиса, word-diff, sticky-заголовком,
  режимом «только hunk'и ↔ весь файл» (`a`), прыжками по изменениям (`[` / `]`),
  горизонтальным скроллом (`h` / `l`) и поиском (`/`, `n` / `N`).
- Русская раскладка шорткатов (по позиции клавиши).

Папка проекта и git-корень определяются по `cwd` окна, из которого нажат хоткей.

## Подключение

В `~/.config/kitty/kitty.conf` (или include-файле):

```conf
map cmd+shift+l kitten /path/to/familiar/plugins/log.py
```

Перезагрузить конфиг: `Cmd+Ctrl+,`. Открыть: `cmd+shift+l`.

## Клавиши

**Список коммитов**

| Клавиша | Действие |
|---|---|
| `↑/↓` | навигация по коммитам |
| `PgUp` `PgDn` | страница вверх/вниз |
| `Home` / `End` | в начало / конец списка |
| `Enter` `→` | открыть изменения коммита |
| `⌘c` | скопировать hash выбранного коммита |
| `f` | `git fetch` — подтянуть удалёнки и перечитать |
| `a` | текущая ветка ↔ все ветки (`--all`) |
| `g` | граф веток вкл/выкл |
| `i` | панель подробностей вкл/выкл |
| `/` | фильтр по subject / hash / автору |
| `q` `Esc` | выход |

**Дерево файлов коммита** (`Enter`/`→` из списка)

| Клавиша | Действие |
|---|---|
| `↑/↓` | навигация по файлам (дифф справа обновляется) |
| `g` / `G` | первый / последний файл |
| `Enter` `Space` | свернуть/развернуть папку |
| `→` `Tab` | перейти в дифф |
| `u` | показать/скрыть «шумные» папки (`.idea`, `node_modules`, …) |
| `⌘c` | скопировать путь файла |
| `←` `Esc` | назад к списку коммитов |

**Дифф коммита** (`→`/`Tab` из дерева)

| Клавиша | Действие |
|---|---|
| `↑/↓` | курсор по строкам диффа |
| `g` / `G` | к началу / концу диффа |
| `Enter` | на разделителе `┈` — раскрыть скрытые строки контекста |
| `[` / `]` | предыдущее / следующее изменение |
| `PgUp` `PgDn` | скролл диффа |
| `h` / `l` | горизонтальный скролл (длинные строки) |
| `a` | режим просмотра: только hunk'и ↔ весь файл целиком |
| `/` `n`/`N` | поиск по диффу и переходы по совпадениям |
| `⌘c` | скопировать выделение (или строку под курсором) |
| `⌘shift+c` | скопировать `путь:строка` |
| `←` `Tab` | назад в дерево |
| `Esc` | снять выделение/поиск → назад в дерево → назад к списку |

**Мышь**: клик по коммиту — выбрать, повторный — открыть; колесо — прокрутка списка.
В диффе — как в review (клик — курсор, клик по `┈` — раскрыть, выделение с зажатым `Shift`).

Статусы файлов и «шумные» папки — как в [review](./review.md). В отличие от review, log —
**просмотр только для чтения**: без аннотаций, скоупов, refresh и открытия в редакторе.

## Работа с Claude Code

Из истории удобно сослать Claude Code на конкретное место в коде:

- `⌘c` кладёт **hash коммита** (в списке), **абсолютный путь** файла (в дереве) или
  **выделение / строку под курсором** (в диффе).
- `⌘shift+c` в диффе кладёт `путь:строка` — абсолютный путь с номером строки под курсором.

Claude Code понимает `путь/до/файла.py:42` как точную ссылку и открывает/правит именно
это место — не нужно пересказывать словами, где смотреть. Навёлся на строку в диффе →
`⌘shift+c` → `Cmd+V` в промт.
